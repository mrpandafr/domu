What it is

Domu is a vector memory engine for AI agents — the first system built as a true vector corpus expansion engine from the ground up. It doesn't just store embeddings. It recalls, enriches, filters, and clusters.

01

L1 — Focus

The moving center. An exponential moving average of recent turns. Never stored, always computed. Drift detection comes for free.

02

L2 — Vault

The expanding knowledge base. Documents indexed by vector, enriched by external calls, retrievable by concentric circles around the focus.

03

L3 — Doors

Fixed semantic categories that never grow. A hit is attached to its nearest door per query — taxonomy cannot inflate, by construction.

"Never embroider reality. If no memory matches, say so. If a fragment is orphaned, admit it. If an external call fails, answer 'I can't'."

— The absolute rule, stamped in every system prompt

The name

Domu (童夢) = child (童) + dream (夢)

A manga by Katsuhiro Ōtomo — the same creator who gave us Akira, which gave its name to the Tamashii Corpus. The story of a child with psychic powers in a housing project. Pure, untamed, unburdened by layers of abstraction.

The name carries the project's soul: a memory system that keeps a child's clarity. Simple. Direct. Four files, not six hundred kilobytes of monolith.

"Domu is the dream. If no occurrence is found, the dream never leaves a void — it responds with an invitation." — JS, 7 July 2026

Wired — Domu nodes by ES

Wired is simply: multiple Domu instances sharing the same Elasticsearch cluster. Each agent runs its own DomuProvider. All share the same vector space. Isolation is by bank_id + scope — enforced in the query, never post-filtered.
